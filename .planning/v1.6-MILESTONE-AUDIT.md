### Integration Bug: Evidence Section Index Mismatch

**Location:** `src/core/evidence.ts` lines 195-197
**Trigger:** `--no-sysinfo` flag on coolify/dokploy platform
**Root cause:** `SECTION_FILENAMES` uses static `EVIDENCE_SECTION_INDICES` constants (SYSINFO=4, DOCKER_PS=5, DOCKER_LOGS=6). When `noSysinfo=true`, `buildEvidenceBatchCommand` skips sysinfo section, shifting docker sections to positions 4 and 5. Parser iterates by position and maps to wrong filenames.
**Impact:** Docker PS output written as "system-info.txt", Docker logs as "docker-containers.txt", "docker-logs.txt" never created.
**Default behavior:** NOT affected. Bug only triggers with optional `--no-sysinfo` flag.
**Fix:** Build SECTION_FILENAMES dynamically based on same flags passed to command builder.
